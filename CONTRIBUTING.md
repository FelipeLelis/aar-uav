# Contribuindo

Obrigado pelo interesse em contribuir com o projeto.

## Fluxo recomendado

1. Abra uma issue descrevendo o problema, melhoria ou experimento.
2. Faça um fork do repositório.
3. Crie uma branch a partir da `main`.
4. Faça a alteração com escopo pequeno e claro.
5. Rode os testes disponíveis:

```bash
npm test
```

6. Abra um pull request para `FelipeLelis/aar`.

## Política de merge

A branch `main` representa a versão principal do projeto. Merges na `main` são responsabilidade do mantenedor, Felipe Lelis.

Pull requests externos são bem-vindos, mas devem passar por revisão antes de qualquer merge.

## Boas práticas

- Mantenha alterações pequenas e revisáveis.
- Não inclua credenciais, tokens, arquivos locais ou saídas geradas desnecessárias.
- Explique decisões técnicas quando alterar regras de simulação.
- Para mudanças visuais, inclua uma descrição clara da tela afetada.
